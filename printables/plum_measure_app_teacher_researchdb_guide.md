# 선생님용 연구DB 운영 방법

## 1. 연구DB는 누가 사용하나요?

연구DB는 선생님이 사용하는 메뉴입니다.  
학생들은 측정과 저장, 이력 확인까지만 합니다.

학생들이 연구DB, 연구앱, 초기화 버튼을 누르지 않도록 안내합니다.

## 2. 연구DB를 누르는 시점

학생들이 계속 측정 중일 때는 연구DB 작업을 하지 않습니다.

연구DB 작업은 아래 시점에 합니다.

- 전체 측정이 끝났을 때
- 한 회차가 완전히 끝났을 때
- 수업이 끝난 뒤 최종 백업할 때

## 3. 백업 순서

1. 모든 학생의 측정이 끝났는지 확인합니다.
2. 학생들이 `이력`에서 저장 여부를 확인하게 합니다.
3. 선생님이 `연구DB` 메뉴를 엽니다.
4. 자료 건수를 확인합니다.
5. `백업 후 초기화`를 누릅니다.
6. 저장 완료 메시지를 확인합니다.
7. 앱 임시자료가 0건인지 확인합니다.

## 4. 버튼별 의미

### 앱자료 백업

앱 안에 임시로 저장된 측정자료를 PC에 CSV 파일로 저장합니다.

- 앱 안 자료는 그대로 남습니다.
- 실제 수업 중에는 학생 측정이 모두 끝난 뒤에만 사용합니다.
- 보통은 `백업 후 초기화`를 사용합니다.

### 백업 후 초기화

앱 안 임시자료를 PC에 저장한 뒤, 앱 안 자료를 비웁니다.

- 반드시 학생 측정을 멈춘 뒤 사용합니다.
- 한 회차가 완전히 끝났을 때 사용합니다.
- 자료가 많아져 앱이 느려질 때 사용합니다.

### 초기화만

PC 백업 없이 앱 안 자료만 지웁니다.

- 실제 수업 중에는 사용하지 않습니다.
- 테스트 자료를 지울 때만 사용합니다.

## 5. 내일 권장 운영 방식

### 수업 시작 전

1. 서버를 켭니다.
2. 학생 접속 주소를 안내합니다.
3. 연구앱에서 `앱 임시자료 0건`인지 확인합니다.
4. PC 연구자료 폴더가 비어 있는지 확인합니다.

### 측정 중

1. 학생들은 측정하고 저장합니다.
2. 학생들은 `이력`에서 저장 확인만 합니다.
3. 선생님도 측정 중에는 `연구DB`를 누르지 않습니다.

### 회차가 끝났을 때

1. 학생들에게 저장을 모두 끝내고 멈추라고 안내합니다.
2. 선생님이 `연구DB`를 엽니다.
3. `백업 후 초기화`를 누릅니다.
4. 앱 임시자료가 0건인지 확인합니다.
5. 다음 회차를 시작합니다.

### 수업 종료 후

1. 마지막으로 `백업 후 초기화`를 실행합니다.
2. `research_exports` 폴더에 CSV가 저장되었는지 확인합니다.
3. 서버 창과 터널 창을 닫습니다.

## 6. 학생들에게 안내할 문장

측정 중 학생들에게 이렇게 안내합니다.

> 연구DB와 연구앱은 선생님 전용입니다. 학생은 누르지 말고, 측정 후 이력에서 저장 여부만 확인하세요.

측정이 모두 끝나면:

> 모든 측정이 끝났으면 저장 여부를 이력에서 확인하세요. 지금부터 선생님이 연구DB 백업을 진행합니다.

## 7. 가장 중요한 주의사항

- 학생들이 저장 중일 때 `백업 후 초기화`를 누르지 않습니다.
- 실제 수업 중 `초기화만`은 누르지 않습니다.
- 자료가 많아질수록 중간 백업을 자주 합니다.
- 회차가 끝난 뒤에는 `백업 후 초기화`로 앱 안 자료를 비웁니다.
